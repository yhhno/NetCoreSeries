关于容器的三个基本概念镜像 容器 仓库
首先演示下docker的使用
docker -v
给大家演示下，docker  怎么样的一个很轻很快
docker pull 一个镜像比如redis
doker pull redis 默认官方最新 

docker images 找到所有的本地的镜像
镜像就是我们打包好的一个镜像 仓库呢是用来存放所有镜像的地方，仓库里面的镜像 ，本地的镜像 本地的镜像需要通过push 到镜像仓库，也可以通过pull把镜像仓库的镜像 拉到本地， 容器是什么呢 容器是基于本地镜像来运行的一个容器
 每一个容器是由docker engine来管理或者来进行隔离性的事情，默认情况下，会使用docker hub来做官方的镜像仓库 镜像拿到本地，通过docker run就可以运行成一个容器，容器运行起来以后，你就可以直接访问了
 docker ps 查看本地运行的容器
